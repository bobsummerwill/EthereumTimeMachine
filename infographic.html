PART 1: CHAIN OF GETHS

A Protocol Bridge Spanning Ethereum's Entire History

THE PROBLEM

Different Ethereum eras speak different "languages" (eth/* protocols). Modern clients can't talk to ancient ones directly.

  eth/68-69 (2025)  <--X-->  eth/60-61 (2015)
        |                          |
    Modern Geth              Frontier Geth
        |                          |
        +----- INCOMPATIBLE! ------+
THE SOLUTION: Protocol Bridge Chain

Chain adjacent versions that share at least ONE common protocol. Each node translates between eras like a relay chain.

PART 2: RESURRECTION PROJECT

Mining Historical Chains Back to Life

THE GOAL

Extend historical Ethereum chains BEYOND their original blocks. Reduce difficulty to CPU-mineable levels for educational testnets.

Frontier lacks EIP-2's -99 multiplier, making it 99x slower to crash difficulty.

DIFFICULTY ALGORITHM (HOMESTEAD EIP-2)
adjustment = max(1 - (timestamp_delta // 10), -99)
new_diff = parent_diff + (parent_diff // 2048) * adjustment

With 1000s timestamp gaps (minimum for max effect):
  adjustment = max(1 - 100, -99) = -99
  Each block reduces difficulty by ~4.83%
